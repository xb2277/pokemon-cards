## 数据采集管道标准化（data_pipeline.py）

### 架构设计
- 创建 `data_pipeline.py` 核心模块，嵌入程序核心
- 管道注册表机制：`@register_method(order=N)` 装饰器注册新方法
- 标准方法接口：id / name / description / source / version / run(dry_run) / order
- 幂等执行：按 name+set_name+card_number+rarity 四元组去重，重复运行只更新不重复插入
- 执行历史追踪：`pipeline_runs` 表记录每次执行的 方法名/状态/统计/时间
- 双入口：CLI (`python data_pipeline.py --list/--run/--run-all/--history`) + API (`/api/pipeline/*`)

- 已验证幂等性：连续执行3次，0重复插入，0重复价格记录
