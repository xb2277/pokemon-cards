
## 当前估值 Bug 修复（2026-06-26）

### 问题
资产看板中"当前估值"始终为 0，因为：
- `get_dashboard_data()` 用 `card_id` 查 `price_records`
- 重构后 `price_records` 改用 `catalog_id` 存储价格
- `cards` 表和 `card_catalog` 表无关联字段

### 修复内容

1. **`cards` 表加 `catalog_id` 字段**
   - `init_db()` 中 CREATE TABLE 和 migration 均加入 `catalog_id INTEGER DEFAULT NULL`
   - 加索引 `idx_cards_catalog`

2. **数据迁移**（`_migrate_cards_catalog_link()`）
   - 按 `(name, set_name)` 匹配，fallback 到 `name` 单独匹配
   - 支持 `set_code` 匹配（处理 set_name 缩写问题）
   - 现有 11 张卡片全部成功关联

3. **`get_dashboard_data()` 和 `get_rankings()` 修复**
   - 优先用 `catalog_id` → `get_catalog_latest_price(catalog_id)` 查价
   -  fallback 到 `price_records(card_id)`（向后兼容）
   - **再加 fallback**：`price_records` 无数据时用 `cards.market_price`

4. **`create_card()` / `update_card()` / `ensure_card_from_catalog()`**
   - 均支持 `catalog_id` 字段

5. **API 修复**
   - `GET /api/dashboard`：admin 账号返回全部数据（不过滤 user_id）

### 结果
- `当前估值` 正确显示：13005.67（11 张卡片全部估值成功）
- 排行榜正常显示涨跌幅

## 新增卡牌流程改造（2026-06-26）

### 目标
用户添加"我的卡牌"时，优先从卡牌数据库搜索选择；搜不到的冷门卡仅存入个人仓库。
卡牌数据库仅管理员可维护。

### 前端改造（app.js + index.html）
1. `index.html` 表单加 `<input type="hidden" name="catalog_id" id="catalogIdField">`
2. `fillFormFromCatalog(item)` 选中数据库卡牌时写入 `catalogIdField`
3. `clearCatalogSelection()` 清除 `catalogIdField.value`
4. `saveCard()` 判断：
   - 有 `catalog_id` 且是新增 → `POST /api/cards/from-catalog`
   - 编辑模式 → `PUT /api/cards/<id>`（保留原 catalog_id）
   - 无 `catalog_id` → `POST /api/cards`（个人冷门卡）

### 后端改造（db.py + app.py）
1. `ensure_card_from_catalog()` 函数签名增加 `quantity, cost_price, condition, notes` 参数
2. INSERT 语句补全 `condition` 字段，VALUES 改用参数而非写死 `1, 0`
3. `POST /api/cards/from-catalog` 接口接收并传递新字段

### 缓存
- index.html: app.js?v=8 → v=9
- price.html: app.js?v=9 → v=10

### 待验证
- 新增时选数据库卡牌 → 提交 → 是否调用 from-catalog 且 quantity/cost_price 正确
- 编辑已关联 catalog 的卡牌 → 是否走 PUT 且保留 catalog_id
- 添加个人冷门卡（不选数据库）→ 是否走 POST /api/cards
